* Overview

This document is the implementation plan for the =acmeid= project: a
small C library, CLI tool, and SQLite loadable extension for minting
and verifying ACME-style identifiers (see [[file:docs/spec.org]],
derived from =acmeid-proposal.org=).

** Goals
- Library: pure, reentrant C core with no I/O.
- CLI: =acmeid mint | verify | batch=, scriptable.
- SQLite extension: =acme_mint_id= (4 arities) and =acme_verify_id= UDFs.
- Recipe: idempotent "spine table" pattern for OTTR-bound ontology
  source databases, replacing CSV round-trips.

** Primary platform
- *Windows / MSYS2 mingw-w64* is the primary build and runtime target.
  Install path convention: =c:/opt/acmeid/acmeid.dll=, loaded via
  =.load c:/opt/acmeid/acmeid= in =/mingw64/bin/sqlite3=.
- Linux and macOS supported in CI; same Makefile and source tree.

** Canonical signature
#+begin_src c
int acme_mint_id(char type,
                 const char *prefix,    /* NULL/"" → no prefix */
                 const char *label,     /* NULL/"" → no slug   */
                 int rand_len,          /* clamp to [2,8]; <=0 → 4 */
                 char *out, size_t out_size);

int acme_verify_id(const char *id);     /* tolerates optional prefix */
#+end_src

Argument order in SQL/CLI mirrors this: =type, prefix, label, length=
— optimised for the RDF/CURIE use case.

** DONE 1.3 Implement new =acme_mint_id= signature
*** DONE Argument validation and normalisation
- =type=: upper-case it; reject if =!isalpha(type)= (return -1).
- =prefix=: NULL/empty → skip; else copy verbatim into =out= (NOT in checksum).
- =label=: NULL/empty → omit slug; else call =acme_slug5=.
- =rand_len=: =<= 0= → 4; else clamp silently to =[2,8]=.
*** DONE Build /core/ string for checksum
- Format: =T_[SLUG]TIMERAND= (prefix excluded).
*** DONE Assemble final output
- =[PREFIX]CORE+CHK= via =snprintf=, all length-checked.
*** DONE Buffer-safety
- Check =out_size= before each write.
- Return =-1= on overflow; set =out[0]='\0'= for safety.
*** Acceptance
- Round-trip test (mint then verify) passes for all 4 call shapes:
  - bare type;
  - type + prefix;
  - type + prefix + label;
  - type + prefix + label + length.

** DONE 1.4 Update =acme_verify_id= for new forms
*** DONE Tolerate optional prefix
- Scan for last =':'= in input; if present, validate suffix only.
- Prefix itself is not checksummed and is not validated structurally.
*** DONE Accept length variants
- With slug: =T_SSSSSTTTTRR…RC= (5 + 4 + rand_len + 1 after =T_=).
- Without slug: =T_TTTTRR…RC= (4 + rand_len + 1 after =T_=).
*** DONE Compute checksum
- Sum =char_value(c)= for every char in the suffix except =_=.
- Verify =sum mod 32 == 0=.
*** Acceptance
- Golden table of 8 IDs (2 arities × 4 random lengths) all verify.
- Tampering any single character invalidates.
- =verify_id("lepus:C_029BGJH65")= returns 1.

** DONE 3.2 Register =acme_mint_id= UDF — four arities
*** DONE Arity 1: =acme_mint_id(type)=
- No prefix, no slug, length 4.
*** DONE Arity 2: =acme_mint_id(type, prefix)=
- With prefix, no slug, length 4.
*** DONE Arity 3: =acme_mint_id(type, prefix, label)=
- With prefix and slug, length 4.
*** DONE Arity 4: =acme_mint_id(type, prefix, label, length)=
- All four parameters.
*** DONE All registrations *omit* =SQLITE_DETERMINISTIC=

  and reuse the same ID across rows. Silent and catastrophic.
- Add a comment in the source explaining why.
*** DONE Argument extraction
- =sqlite3_value_text(arg)= for strings; treat NULL and empty-string as "absent".
- =sqlite3_value_int(arg)= for length; clamping happens in the core.
*** DONE Error handling
- =sqlite3_result_error= with a clear message on bad type.
- =sqlite3_result_text= with =SQLITE_TRANSIENT= on success.
*** Acceptance
- All four call shapes return well-formed IDs:
  - =select acme_mint_id('C')=
  - =select acme_mint_id('C','lepus:')=
  - =select acme_mint_id('C','lepus:','Pitch 1.5 mm')=
  - =select acme_mint_id('C','lepus:','Pitch',6)=

** DONE 3.3 Register =acme_verify_id= UDF (arity 1)
*** DONE Mark =SQLITE_DETERMINISTIC | SQLITE_INNOCUOUS=

* Daily-workflow payoff

Once installed at =c:/opt/acmeid/acmeid.dll=, the existing org-babel
pattern extends naturally:

#+begin_example
:prologue ".load c:/opt/acmeid/acmeid\n.load c:/opt/regex0\n.load c:/opt/sqlean-win-x64/crypto"
#+end_example

Three extensions, one prologue, all loaded into =/mingw64/bin/sqlite3=.
The =fastener-queries.org= recipes drop their CSV round-trips on the
next session.
