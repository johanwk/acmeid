#+title: ACME Localname ID --- v1.1

*Scope:* Localname portion of enterprise IRIs, for intranet RDF as the
canonical data format. *Form:* one fixed pattern with three optional
elements (prefix, slug, random length).

* v1.1 changes (was v1.0)
:PROPERTIES:
:CUSTOM_ID: v1-1-changes
:END:
The v1.0 syntax was =TYPE_SLUG5TIME4RAND4CHK= -- exactly 16 characters
in the localname. v1.1 keeps the v1.0 spelling as one possible shape
but adds three orthogonal options so a single library and a single
SQL UDF can cover all the cases we actually see in practice:

- *Optional prefix.* The localname may be prefixed with a CURIE-style
  =prefix:= (e.g. =lepus:=). The prefix is copied through =mint= and
  tolerated by =verify=, but is *not* part of the checksum. This
  matches the way ids appear in our RDF (always with a prefix) and in
  our SQL spine tables (sometimes stored bare, sometimes with the
  prefix already attached).
- *Optional slug.* When no human label is available, the 5-letter slug
  may be omitted. The remainder still verifies. =T_TTTTRRRRC= (11
  chars) is now a legal shape.
- *Configurable random width.* =RAND= may be any width in =[2,8]= (10
  to 40 bits of entropy). Values =<= 0= are interpreted as "default"
  (=4=); values outside =[2,8]= are silently clamped. The default of
  4 reproduces the v1.0 throughput and collision properties.

The canonical four-arity signature -- used identically in the C
library, the CLI, and the SQLite UDF -- is:

#+begin_example
acme_mint_id(type)
acme_mint_id(type, prefix)
acme_mint_id(type, prefix, label)
acme_mint_id(type, prefix, label, length)
#+end_example

Argument order is /type, prefix, label, length/ -- optimised for the
CURIE use case.

* Windows / MSYS2 build appendix
:PROPERTIES:
:CUSTOM_ID: msys2-build
:END:
The primary build and runtime target is *Windows / MSYS2 mingw-w64*;
Linux and macOS are also supported.

- OS detection uses =uname -s=:
  =Linux*= -> =acmeid.so=,
  =Darwin*= -> =acmeid.dylib=,
  =MINGW*=, =MSYS*=, =CYGWIN*= -> =acmeid.dll=.
- Install convention on MSYS2 is =c:/opt/acmeid/acmeid.dll=, alongside
  =/mingw64/bin/sqlite3=.
- Load mechanics in SQLite: =.load c:/opt/acmeid/acmeid= -- *no =.dll=
  suffix* (SQLite appends it) and *forward slashes are fine* (SQLite
  normalises them).
- The DLL is built with =-static-libgcc= so it has no dependency on
  =libgcc_s_seh-1.dll= when loaded from a non-MSYS sqlite shell.
  =bcrypt.dll= (for =BCryptGenRandom=) is a system DLL.

* 1) Syntax
:PROPERTIES:
:CUSTOM_ID: syntax
:END:
#+begin_example
[<PREFIX>:]<TYPE>_[<SLUG>]<TIME><RAND><CHK>
#+end_example

- =PREFIX= --- /v1.1, optional./ CURIE-style prefix terminated by =:=.
  Copied through =mint= and tolerated by =verify=. *Not part of the
  checksum.*
- =TYPE= --- 1 uppercase letter owned by a domain (e.g., =C= concept, =D= document, =I= invoice, =A= asset...).
- =_= --- underscore separator.
- =SLUG= --- /v1.1, optional./ 5 lowercase letters from an English label (ASCII a--z; see §2).
- =TIME= --- 4 Crockford Base32 chars = *days since 2020-01-01 UTC*, left-padded (sortable by day).
- =RAND= --- /v1.1: configurable width =N= in =[2,8]= (default 4), giving =5N= bits of entropy./
- =CHK= --- 1 Crockford Base32 *check symbol* (mod-32 sum, see §2).

*Alphabet (Crockford Base32):* =0123456789ABCDEFGHJKMNPQRSTVWXYZ= (no I, L, O, U).

*Regex (validation):*

The v1.0 regex still matches the "with slug, RAND=4" shape:

#+begin_example
^[A-Z]_[a-z]{5}[0-9A-HJKMNP-TV-Z]{4}[0-9A-HJKMNP-TV-Z]{4}[0-9A-HJKMNP-TV-Z]$
#+end_example

The v1.1-permissive regex (optional prefix, optional slug, RAND in
[2,8]):

#+begin_example
^([A-Za-z][\w-]*:)?[A-Z]_(?:[a-z]{5})?[0-9A-HJKMNP-TV-Z]{4}[0-9A-HJKMNP-TV-Z]{2,8}[0-9A-HJKMNP-TV-Z]$
#+end_example

Implementations should regard the regex as a /shape/ check only;
authoritative validation is the checksum (=acme_verify_id=).

* 2) Generation rules
:PROPERTIES:
:CUSTOM_ID: generation-rules
:END:
- *SLUG(5):* Take =skos:prefLabel= [cite/t:@en] → NFKD → ASCII → lowercase → keep =a–z= → first 5; if <5, pad with =x=. Examples: "Employment contract" → =emplo=; "ISO 50001" → =isoxx=; "Ångström" → =angst=.

- *TIME(4):* =days = floor((nowUTC − 2020-01-01T00:00:00Z)/86400)= → encode in Crockford Base32, width=4 (left-pad with =0=).

- *RAND(4):* 20 random bits → Crockford Base32, width=4.

- *CHK(1):* Let =core = "<TYPE>_<SLUG><TIME><RAND>"=. Map each Crockford char to its value (ignore =_=). Compute =s = Σ values(core)=. =CHK = ALPH[(32 − (s mod 32)) mod 32]=. *Verification:* =Σ values(<full ID>) mod 32 == 0=.

* 3) Collision handling & throughput
:PROPERTIES:
:CUSTOM_ID: collision-handling-throughput
:END:
- Random space per *day*: 2²⁰ ≈ 1,048,576.
- Enforce a *unique index* on the localname; on conflict, regenerate =RAND= and recompute =CHK= (retry loop).
- For /very large same-day backfills/, run multiple workers or spread across days; if needed, temporarily use a sequenced payload under a controlled "lease" subspace (v2 policy).

* 4) Policy & governance (minimal)
:PROPERTIES:
:CUSTOM_ID: policy-governance-minimal
:END:
- *Non-reuse:* IDs are never reassigned.
- *Minting:* via API/CLI libs; domains own the =TYPE= letter.
- *Search-before-mint (recommended):* the API should show near-duplicates to encourage reuse.
- *Statuses:* model in RDF (=provisional | canonical | deprecated=)---*status is metadata*, never part of the ID.

*Python (single file):*

#+begin_src python
import os, time, unicodedata, re
ALPH = "0123456789ABCDEFGHJKMNPQRSTVWXYZ"; VAL={c:i for i,c in enumerate(ALPH)}
def slug5(s):
    s=unicodedata.normalize("NFKD",s).encode("ascii","ignore").decode().lower()
    s=re.sub(r"[^a-z]+","",s); return (s+"xxxxx")[:5]
def b32(n,w):
    s="0" if n==0 else ""; 
    while n>0: n,r=divmod(n,32); s=ALPH[r]+s
    return s.rjust(w,"0")
def randb32(w):
    bits=w*5; n=int.from_bytes(os.urandom((bits+7)//8),"big") & ((1<<bits)-1)
    return b32(n,w)
def chk(core):
    s=sum(VAL[c] for c in core if c!="_"); return ALPH[(32-(s%32))%32]
def mint(t, label_en):
    days=int((time.time()-1577836800)//86400)
    core=f"{t.upper()[0]}_{slug5(label_en)}{b32(days,4)}{randb32(4)}"
    return core+chk(core)

#+end_src

*Emacs Lisp (compact):*

#+begin_src elisp
(defun acmeid--slug5 (s)
  (let* ((s (downcase (subst-char-in-string ?ø ?o
             (subst-char-in-string ?å ?a (subst-char-in-string ?æ ?e s)))))
         (s (replace-regexp-in-string "[^a-z]" "" s)))
    (substring (concat s "xxxxx") 0 5)))
(defconst acmeid-b32 "0123456789ABCDEFGHJKMNPQRSTVWXYZ")
(defun acmeid--b32 (n w)
  (let ((s ""))
    (while (> n 0)
      (setq s (concat (string (aref acmeid-b32 (% n 32))) s)
            n (/ n 32)))
    (setq s (if (string-empty-p s) "0" s))
    (concat (make-string (max 0 (- w (length s))) ?0) s)))
(defun acmeid--randb32 (w)
  (let* ((bits (* 5 w)) (bytes (ceiling bits 8.0))
         (n 0))
    (dotimes (_ bytes) (setq n (+ (* n 256) (random 256))))
    (setq n (logand n (1- (ash 1 bits))))
    (acmeid--b32 n w)))
(defun acmeid--chk (core)
  (let* ((val (lambda (c) (cl-position c acmeid-b32)))
         (sum (apply #'+ (mapcar (lambda (c) (if (= c ?_) 0 (funcall val c)))
                                 (string-to-list core)))))
    (string (aref acmeid-b32 (mod (- 32 (mod sum 32)) 32)))))
(defun acmeid-mint (type label)
  (let* ((days (/ (- (float-time) (float-time (encode-time 0 0 0 1 1 2020 0 0 0))) 86400))
         (core (format "%c_%s%s%s"
                       (upcase (aref type 0))
                       (acmeid--slug5 label)
                       (acmeid--b32 (floor days) 4)
                       (acmeid--randb32 4))))
    (concat core (acmeid--chk core))))
#+end_src

* Roll-out notes
:PROPERTIES:
:CUSTOM_ID: roll-out-notes
