#+title: acmeid recipes -- spine tables for OTTR
#+author: johanw
#+date: 2026

* Why this exists

The historical workflow for our ontology source databases was:

1. =sqlite3= computes the rows.
2. The script exports CSV.
3. A separate Python/Emacs step mints identifiers, writes them back as
   another CSV.
4. OTTR templates read the CSVs.

That round-trip exists only because SQLite didn't know how to mint our
identifiers.  Once the =acmeid= extension is loaded, the spine table
is the registry, =acme_mint_id= is the registrar, and OTTR templates
read directly from the canonical SQL.  No CSV ferry, no risk of
re-minting on re-run.

The pattern in one sentence: /each spine table has a CHECK constraint
that the id verifies, and is populated by an idempotent insert that
mints only for rows that aren't already there./

* Loading the extension

** MSYS2 / Windows (canonical for this project)

#+begin_example
# Interactive sqlite3 in /mingw64/bin:
sqlite> .load c:/opt/acmeid/acmeid

# org-babel sqlite block:
:prologue ".load c:/opt/acmeid/acmeid"
#+end_example

Three small Windows notes:

1. Forward slashes are fine -- SQLite normalises them.
2. /No =.dll= suffix needed/ -- SQLite appends it automatically.
3. =-static-libgcc= is baked into the DLL, so =libgcc_s_seh-1.dll= is
   not required at load time.  =bcrypt.dll= is a system DLL.

** Linux / macOS

#+begin_example
sqlite> .load /usr/local/lib/acmeid
#+end_example

** When =.load= refuses

Some sqlite builds (notably the one shipped inside CPython on many
systems) are compiled with =SQLITE_OMIT_LOAD_EXTENSION=.  Use a
build that supports loading -- on MSYS2, =/mingw64/bin/sqlite3=
works out of the box; for Python, =pysqlite3-binary= does.

* The spine-table pattern

#+begin_src sql
CREATE TABLE dim_pitch (
    id    TEXT PRIMARY KEY CHECK (acme_verify_id(id)),
    label TEXT NOT NULL UNIQUE
);

-- Idempotent insert: mint only for rows that aren't already there.
INSERT INTO dim_pitch(id, label)
SELECT acme_mint_id('C','ex:',v.label), v.label
FROM (VALUES
    ('Pitch 1.0 mm'),
    ('Pitch 1.25 mm'),
    ('Pitch 1.5 mm')
) AS v(label)
WHERE NOT EXISTS (SELECT 1 FROM dim_pitch p WHERE p.label = v.label);
#+end_src

Running the same =INSERT= a second time adds zero rows; the spine is
authoritative and the ids are stable across runs.
